# Full VI Manual Outline

Use this reference when planning or rendering a complete HTML brand manual. The final manual must visibly cover all 21 chapters below. Do not omit chapters.

## Completion Rules

- Render all 21 top-level chapters in the HTML manual, in this order.
- Render every numbered subsection as visible content, a specification table row, a card, an example panel, or a clearly labeled "Not applicable / 不适用" item with a short reason.
- Do not hide unsupported sections. If a touchpoint is not relevant to the brand, keep the section and mark it as "Not applicable for this brand stage" with a reason.
- For full VI delivery, include a sticky or visible table of contents that links to all 21 chapters.
- Each chapter should include practical guidance, not only a heading. Use rules, examples, tokens, prompts, ratios, do/don't notes, or asset slots.
- Use generated images for the most important touchpoints. For sections without generated images, include a production-ready image prompt, layout description, or placeholder panel clearly marked as pending asset generation.
- The final QA must check chapter coverage before delivery.

## HTML Rendering Guidance

Use a structure that makes the manual complete but still readable:

- A hero/cover with brand name, positioning, and version.
- A fixed or sticky side navigation with all 21 chapters.
- Chapter sections with numbered headings and short executive summaries.
- Subsection grids or tables for dense specification content.
- Copy buttons for color values, tokens, file naming patterns, and prompts.
- Image panels for logo assets and generated applications.
- "Asset pending" panels for approved sections that still need more generated imagery.
- A final coverage checklist that lists all 21 chapters and marks each as complete, not applicable, or pending asset generation.
